Phase 12 方案：顶部新增“快捷键盘开关”按钮（客户端页）
Summary
在当前 Android 客户端架构下（MainShellActivity + WebView 加载 terminal_client.html），新增一个位于顶部右侧（Settings 旁）的按钮，用来打开/收起网页内快捷键盘栏 #toolbar。
已锁定默认行为：

快捷键盘默认展开。
开关状态不跨重启记忆。
仅影响客户端页 terminal_client.*，不改网页端 terminal.*。
Important Changes / Interfaces / Types
后端 API/WS 协议：无变更。
Android 原生新增一个顶部按钮控件：
btn_toggle_quick_toolbar（ImageButton）
Native -> Web 新增控制函数调用（evaluateJavascript）：
window.__setQuickToolbarVisible(visible: boolean)
window.__toggleQuickToolbarVisible()（可选）
Web 端 terminal_client.js 新增页面内状态：
let quickToolbarVisible = true（默认 true，不持久化）
Current Baseline (基于现状确认)
顶栏已有：
左：btn_open_sessions_drawer
右：btn_open_settings
状态：shell_status_text
App 入口加载客户端页：
terminal_client.html?v=19
客户端快捷键栏是网页 DOM：
terminal_client.html 中 <div id="toolbar">...
重要流程约束：
资源变更后必须执行 npm run android:sync，否则 APK 仍会打入旧 android/app/src/main/assets/public/*。

C. 客户端页 JS/CSS 接口

Behavior Specification (决策锁定)
默认进入 Terminal：快捷键栏展开。
点击顶部“快捷键盘按钮”：
展开 -> 收起
收起 -> 展开
切到 Settings 再回 Terminal：
本轮不记忆到持久化，但在当前 Activity 生命周期内保持当前状态（推荐）
冷启动后恢复默认展开
不影响：
输入法开关逻辑
会话切换逻辑
快捷键按键映射与 WS 协议
Test Cases and Scenarios
顶栏按钮可见性
Terminal 页面显示“快捷键盘按钮”
Settings 页面隐藏该按钮
开关行为
初始展开（默认）
点一次收起，终端可视高度增加，sendResize 生效
再点一次展开，快捷键栏恢复
交互回归
单击/双击输入法逻辑不回归
Esc/Tab/Ctrl-C/Enter/方向键在展开时仍正常
收起时不会误触发其他 UI 错位
生命周期
Terminal -> Settings -> Terminal 不崩溃
当前 Activity 内状态保持（若已收起，返回 Terminal 仍收起）
打包发布链路
npm run android:sync 后，terminal_client.html 版本号与 terminal_client.html 一致
真机安装后，顶部按钮生效
