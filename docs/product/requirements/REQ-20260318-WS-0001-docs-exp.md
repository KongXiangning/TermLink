REQ-WS-0001 工作区文件浏览与 Diff 查看
1. 基本信息

编号：REQ-WS-0001

名称：工作区文件浏览与 Diff 查看

状态：Draft

优先级：High

目标版本：V1

所属模块：Codex 会话 / Workspace

目标平台：Android 优先

2. 背景

用户希望在 Codex 会话中直接查看当前项目目录下的文件，而不必频繁通过终端命令查看目录、读取文件或执行 Git diff。

用户的核心诉求包括：

在线查看指定目录下的文件

查看目录结构与文件列表

查看文件内容

当当前目录处于 Git 仓库范围内时，查看文件是否有变动

对已打开文件选择 Diff 模式查看差异

在创建 Codex 会话时，为 Codex Workspace Path 提供目录选择能力，减少手输路径出错

本功能第一阶段仅要求在 Codex 会话场景中可用，且工作区目录应绑定为打开该 Codex 会话时所使用的目录。
Codex Workspace Path 所指向的目录范围为当前服务端宿主机上的目录，而非 Android 设备本地目录。

3. 目标

本需求的目标是为 Codex 会话提供一个工作区查看能力，使用户可以在移动端直接完成以下操作：

在创建 Codex 会话时选择工作区目录

浏览当前会话工作区下的目录树与文件列表

打开并查看文本文件内容

查看 Git 变更状态

在文件内容视图与 Diff 视图之间切换

在 Android 端通过独立完整的工作区界面使用该能力

4. 范围
4.1 本期范围

本期必须支持：

仅在 Codex 会话下提供工作区功能入口

在创建 Codex 会话时，为 Codex Workspace Path 提供目录选择能力

允许用户手动输入 Codex Workspace Path，也允许通过目录选择器回填

工作区根目录绑定为打开该 Codex 会话时所使用的目录

展示工作区根目录内的完整文件系统视图

支持确定一个默认进入目录

默认进入目录优先为 DOCS 或 docs；若不存在则进入工作区根目录

默认显示隐藏文件，并提供开关

支持目录展开、折叠与文件列表查看

支持打开文本文件查看内容

对文本文件按文件规模提供不同查看模式

若工作区位于 Git 仓库范围内，显示文件变更状态

对已打开文件提供 Diff 模式

Diff 模式采用统一文本 Diff

Android 端提供进入独立工作区界面的能力

4.2 非本期范围

本期不包含：

文件编辑

文件删除、移动、重命名、上传、下载

多标签文件查看

左右分栏 Diff

非文本文件内容预览

聊天输入框中的 @文件补全

跟随终端 cd 实时改变工作区根目录

5. 术语定义
5.1 工作区根目录

当前 Codex 会话绑定的目录。
本期中，该目录固定为打开 Codex 会话时所使用的目录。

5.2 默认进入目录

用户首次打开工作区时默认展示的目录。
该目录必须位于工作区根目录内。未显式配置时，由系统按规则自动选择。
用户进入工作区后，可继续浏览并切换到工作区根目录内的其他目录，但该行为不改变默认进入目录的判定规则。

5.3 Diff 模式

针对当前已打开文件展示统一文本差异的查看模式，不是左右分栏 Diff。

5.4 完整预览

对适合直接展示的小型文本文件，一次性返回全文并在工作区页面内直接显示的查看方式。

5.5 截断预览

对可作为文本查看但不适合一次性全文返回的文件，仅返回前一部分内容，并明确告知已截断的查看方式。

5.6 分段查看

对较大的文本文件，不一次性返回全文，而是按段读取和展示的查看方式。

5.7 受限查看

对超大文本文件，仅允许查看头部或尾部等受限片段，并引导用户通过终端进一步查看的查看方式。

6. 用户故事
US-001

作为 Codex 会话用户，我希望直接看到当前会话目录下的文件树，这样我不需要在终端里反复执行目录查看命令。

US-002

作为 Codex 会话用户，我希望点击文件后直接查看内容，这样我可以快速核对文档、配置和代码。

US-003

作为 Codex 会话用户，我希望看到哪些文件发生了 Git 变更，这样我可以知道当前工作区有哪些改动。

US-004

作为 Codex 会话用户，我希望打开文件后能在内容模式和 Diff 模式之间切换，这样我可以直接查看当前文件的差异。

US-005

作为 Android 用户，我希望在 Codex 会话中进入一个独立完整的工作区界面，这样在手机上浏览文件会更清晰。

US-006

作为创建 Codex 会话的用户，我希望在填写 Codex Workspace Path 时可以直接选择目录，这样可以减少路径输入错误并更快创建会话。

7. 功能需求
7.1 适用范围与会话绑定
FR-001

本功能本期必须仅对 Codex 会话开放。

FR-002

系统必须为每个 Codex 会话确定一个唯一的工作区根目录。

FR-003

本期工作区根目录必须等于打开该 Codex 会话时所使用的目录。

FR-004

工作区功能的所有文件访问范围必须限制在该工作区根目录内。

FR-005

本期工作区根目录不得因终端中的目录切换而动态变化。

7.2 创建会话时的工作区路径选择
FR-006

在创建 Codex 会话时，系统必须提供 Codex Workspace Path 输入能力。

FR-007

Codex Workspace Path 必须同时支持以下两种输入方式：

手动输入

通过目录选择器选择并回填

FR-008

目录选择器必须支持用户选择当前服务端宿主机上的目录，并将选定目录路径回填到会话创建表单。

FR-009

本期目录选择能力允许用户在产品允许的完整路径范围内选择目录，不限制为预设业务目录。

FR-010

在提交创建会话前，系统必须至少校验：

路径非空

路径存在

路径为目录而非文件

FR-011

创建会话成功后，系统必须将创建时使用的 Codex Workspace Path 作为该会话的工作区根目录，并在后续恢复该会话时继续使用该固定值。

7.3 默认进入目录
FR-012

系统必须支持为工作区确定一个默认进入目录，该目录需位于工作区根目录内。

FR-013

未显式指定默认进入目录时，系统必须按以下顺序自动选择：

DOCS

docs

工作区根目录

FR-014

默认进入目录仅影响首次进入工作区时的展示位置，不得限制用户继续浏览工作区根目录内的其他目录。

7.4 目录树与文件列表
FR-015

系统必须支持展示工作区根目录内的目录结构与文件列表。

FR-016

系统必须支持目录展开与折叠。

FR-017

系统必须支持完整文件系统视图，而非仅显示文档类文件。

FR-018

系统默认必须显示隐藏文件，并提供开关用于控制显示或隐藏隐藏文件。

FR-019

系统必须支持刷新当前目录内容。

7.5 文件内容查看
FR-020

系统必须支持打开文本文件并展示其内容。

FR-021

用户打开文件后，系统必须默认进入内容查看模式。

FR-022

系统必须支持对当前已打开文件重新加载内容。

FR-023

对于二进制文件或不支持文本查看的文件，系统必须提供明确的不可预览提示。

FR-024

系统必须根据文本文件规模，采用不同的查看模式，而不是一律全文返回。

FR-025

对于适合直接展示的小型文本文件，系统必须支持完整预览。

FR-026

对于中等文本文件，系统必须支持截断预览，并明确提示当前内容已截断。

FR-027

对于截断预览的文本文件，系统应支持继续加载更多内容。

FR-028

对于较大的文本文件，系统必须支持分段查看，而不是一次性返回全文。

FR-029

对于超大文本文件，系统必须提供受限查看模式；该模式至少应支持查看头部或尾部片段中的一种。

FR-030

当系统无法在工作区内高效展示大文件时，必须给出明确提示，并允许用户转而通过终端进一步查看。

FR-031

文件查看区域必须显示当前文件的基本信息，至少包括文件名与相对路径。

7.6 Git 状态与 Diff
FR-032

若工作区根目录位于某 Git 仓库内部，或其本身为 Git 仓库根，系统必须识别该 Git 仓库。

FR-033

在 Git 仓库范围内，系统必须展示文件的 Git 变更状态。

FR-034

文件列表中应显示至少以下状态类型：

Modified

Added

Deleted

Renamed

Untracked

FR-035

若当前工作区不在 Git 仓库范围内，系统必须提供明确状态提示，但不得影响文件浏览与文件内容查看。

FR-036

用户打开文件后，必须可以在“内容模式”和“Diff 模式”之间切换。

FR-037

Diff 模式必须为统一文本 Diff。

FR-038

系统必须支持对当前已打开文件重新获取 Diff 结果。

FR-039

若当前文件无变更，Diff 模式必须显示明确空状态。

FR-040

若当前文件未被 Git 跟踪或不适合生成有效 Diff，系统必须给出明确反馈，不得返回误导性结果。

FR-041

系统对 Diff 输出也必须提供保护机制，避免因过大输出导致界面明显卡顿或无响应。

7.7 Android 端交互
FR-042

Android 端必须为 Codex 会话提供进入工作区的明确入口。

FR-043

进入工作区后，必须呈现一个独立、完整、适合移动端操作的工作区界面。

FR-044

在该界面中，用户必须能够完成目录浏览、文件打开、内容查看与 Diff 查看。

8. 交互与界面要求
UX-001

手机端界面必须优先适配窄屏与触控操作。

UX-002

Diff 查看不得采用左右分栏形式作为本期默认方案。

UX-003

用户必须能清楚识别当前处于内容模式还是 Diff 模式。

UX-004

隐藏文件显示开关必须可见且能即时生效。

UX-005

当目录为空、文件不可预览、当前目录不在 Git 仓库中、文件无变更时，界面必须提供明确提示。

UX-006

当文件处于截断预览、分段查看或受限查看模式时，界面必须明确告知当前查看模式及其限制。

UX-007

工作区界面应尽量减少用户在手机端进行横向滚动或多面板并列操作的负担。

UX-008

在创建会话界面中，Codex Workspace Path 的输入入口与目录选择入口必须清晰可见。

9. 安全与约束
NFR-001

系统必须保证所有路径访问均受工作区根目录约束。

NFR-002

系统必须防止通过相对路径、符号链接或等价方式访问工作区根目录外部文件。

NFR-003

系统不得允许超出工作区范围的任意文件读取。

NFR-004

系统必须对文件读取失败、目录读取失败、Git 查询失败等情况返回明确错误或状态提示。

NFR-005

系统不得因为目录选择器的引入而绕过会话创建时的路径校验逻辑。

NFR-006

系统必须兼容旧会话恢复场景：若旧会话缺失工作区根目录但存在原始 cwd，可按该 cwd 完成一次懒初始化；若两者均不可用，则必须禁用工作区入口并给出明确提示。

10. 性能要求
NFR-007

目录浏览应支持逐级展开，避免一次性加载大型目录树。

NFR-008

常规目录列表打开、文本文件打开和单文件 Diff 查看应达到移动端可接受的响应速度。

NFR-009

打开较大目录、较大文本文件或查看 Diff 时，不应导致整个工作区界面明显卡死。

NFR-010

大文本文件的查看应优先采用分段读取、截断返回或受限查看等策略，避免一次性返回全文。

11. 兼容性要求
NFR-011

本期首要交付平台为 Android，但服务端能力应具备后续复用到其他客户端的可能性。

NFR-012

本功能不得破坏现有 Codex 会话流程。

12. 验收标准
AC-001

用户进入创建 Codex 会话界面后，可以通过手动输入或目录选择器填写 Codex Workspace Path。

AC-002

选择目录后，系统会将所选目录路径正确回填到会话创建表单。

AC-003

提交创建前，系统会对 Codex Workspace Path 做基本有效性校验。

AC-004

用户进入 Codex 会话后，可以通过明确入口进入工作区界面。

AC-005

工作区根目录等于打开该 Codex 会话时所使用的目录。
创建后再次进入该会话时，工作区根目录不因终端中的 cd 或会话恢复而变化。

AC-006

工作区首次打开时，默认进入 DOCS、docs 或根目录中的一个有效目录。

AC-007

用户可以浏览完整文件系统视图，而不是仅浏览文档类文件。

AC-008

隐藏文件默认可见，关闭开关后立即隐藏。

AC-009

用户可展开目录并查看文件列表。

AC-010

点击文本文件后，可进入内容模式查看文件内容。

AC-011

对于中等文本文件，界面能明确提示已截断，并可继续加载更多内容。

AC-012

对于较大的文本文件，系统可按段查看内容，而不是一次性返回全文。

AC-013

对于超大文本文件，系统可提供受限查看，并提示用户通过终端进一步查看。

AC-014

打开文件后，用户可切换到 Diff 模式，再切回内容模式。

AC-015

在 Git 仓库范围内，文件列表可以显示文件变更状态。

AC-016

对于无变更文件，Diff 模式显示明确空状态。

AC-017

对于非 Git 目录，界面给出明确状态提示，且文件浏览与文件内容查看仍可正常使用。

AC-018

用户无法通过本功能访问工作区根目录以外的文件。

AC-019

对于缺失 workspaceRoot 的旧会话，若存在原始 cwd，系统可完成一次懒初始化并正常打开工作区。

AC-020

对于既缺失 workspaceRoot 又缺失 cwd 的旧会话，工作区入口被禁用，并向用户显示明确提示。
