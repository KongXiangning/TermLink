# CR-20260324-1545-req-init

## 1. 变更意图（Compact Summary）

- 背景：Android 会话管理当前高度依赖实时拉取服务端 sessions 列表，网络慢或请求失败时会导致界面无信息可看，用户无法先看到已有会话快照。
- 目标：新增一条正式需求，定义“App 本地缓存远端 profile 的已有 sessions 列表并在会话管理页中优先回显”的产品边界、验收标准和后续实施方向，并明确创建弹窗本身不承担已有 sessions 展示职责。
- 本次边界：仅完成需求立项、主线索引同步和 CR 建档，不包含任何运行时代码实现。

## 2. 实施内容（What changed）

1. 新增 `REQ-20260324-session-list-local-cache`，定义 Android 会话列表本地缓存、离线回显、刷新覆盖和缓存隔离边界。
2. 将该需求同步加入 `REQUIREMENTS_BACKLOG`、`PRODUCT_REQUIREMENTS`、`ROADMAP` 与 `CHANGELOG_PROJECT`。
3. 为本次需求立项新增独立 draft CR，并补入变更记录索引。

## 7. 风险与注意事项

1. 本地缓存只能改善可见性和抗弱网体验，不能替代服务端作为权威数据源。
2. 后续实现必须明确缓存隔离键和过期提示，否则容易造成跨服务端串数据或用户误判“缓存即最新”。
